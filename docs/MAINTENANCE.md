# Сопровождение и передача на поддержку

## Как менять схему БД (миграции)
Схема создаётся в `db.py` (`SCHEMA`, `init_db`). Для изменения:
1. Добавьте новый DDL в `SCHEMA` (идемпотентно, `IF NOT EXISTS`).
2. Для изменения существующих таблиц заведите простую систему версий:
   таблица `meta(key,value)` с `schema_version` и последовательные шаги
   `ALTER TABLE` в функции `migrate()` вызываемой из `init_db()`.
3. Данные пользователя — в файле `teacher_journal.db` (приватный каталог).

## Как добавить экран
1. Создайте `ui/screen_x.py` (класс `XScreen(Screen)`) и `ui/screen_x.kv`.
2. `Builder.load_file(...)` в модуле экрана.
3. Зарегистрируйте в `main.py`: `sm.add_widget(XScreen(name="x"))`.
4. Кнопку перехода добавьте в `screen_home.kv`.

## Известные ограничения
- **Фоновые уведомления Android** ограничены ОС (Doze, точные будильники).
  Основной механизм напоминаний — проверка при открытии приложения; `plyer`
  уведомления вторичны. Для надёжного фона потребуется нативный сервис/
  AlarmManager (вне текущего объёма).
- **PDF-экспорт** использует `reportlab`, только если он установлен; иначе
  автоматически откат к `.txt`. Чтобы PDF работал в APK — добавьте `reportlab`
  в `requirements` (buildozer.spec и requirements.txt).

## Точки расширения
- Новые отчёты — в `services/reports.py`.
- Новые типы диаграмм — в `services/charts.py`.
- Формат `.ics` — в `services/ics_export.py`.

## Миграция на другой фреймворк
Вся бизнес-логика изолирована в `services/` и `db.py` и не зависит от Kivy
(кроме `charts.py`). При переносе UI переиспользуйте эти модули как есть.
